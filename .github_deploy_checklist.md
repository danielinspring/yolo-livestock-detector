# GitHub Deployment Checklist

Use this checklist before pushing to GitHub and deploying to GPU server.

## üìã Pre-Push Checklist (Local Machine)

- [ ] **Remove sensitive data**
  - [ ] No API keys, passwords, or secrets in code
  - [ ] Check for `.env` files (should be in `.gitignore`)
  - [ ] Review all config files

- [ ] **Verify .gitignore**
  - [ ] Virtual environment excluded (`venv/`)
  - [ ] Dataset images excluded (`.jpg`, `.png`, etc.)
  - [ ] Model weights excluded (`.pt` files)
  - [ ] Results directory excluded (`results/`)

- [ ] **Check file sizes**
  - [ ] No files > 100MB (GitHub limit)
  - [ ] Total repo size reasonable (<1GB)
  - [ ] Run: `find . -type f -size +50M | grep -v ".git"`

- [ ] **Test locally**
  - [ ] Scripts run without errors
  - [ ] Requirements.txt is complete
  - [ ] No hard-coded paths (use relative paths)

- [ ] **Documentation**
  - [ ] README.md is up to date
  - [ ] DEPLOYMENT.md has correct instructions
  - [ ] Requirements files are current

## üöÄ Push to GitHub

- [ ] Initialize git (if new repo): `git init`
- [ ] Stage files: `git add .`
- [ ] Commit: `git commit -m "Descriptive message"`
- [ ] Add remote: `git remote add origin <URL>`
- [ ] Push: `git push -u origin main`

## üñ•Ô∏è GPU Server Setup

- [ ] **Server prerequisites**
  - [ ] NVIDIA driver installed
  - [ ] CUDA installed (11.8 or 12.1)
  - [ ] Python 3.8-3.11 available
  - [ ] Git installed
  - [ ] Sufficient storage (50GB+)

- [ ] **Clone and setup**
  - [ ] Clone repo: `git clone <URL>`
  - [ ] Run setup: `./setup_gpu.sh`
  - [ ] Verify GPU detected: `nvidia-smi`
  - [ ] Verify PyTorch CUDA: `python -c "import torch; print(torch.cuda.is_available())"`

- [ ] **Upload data**
  - [ ] Transfer images to server
  - [ ] Verify data integrity
  - [ ] Check file permissions

- [ ] **Preprocess and train**
  - [ ] Run preprocessing: `python scripts/preprocess_data.py`
  - [ ] Split dataset: `python scripts/split_data.py`
  - [ ] Verify dataset: `python scripts/check_setup.py`
  - [ ] Start training: `python scripts/train.py`

## üîÑ Ongoing Updates

- [ ] **Update code**
  - [ ] Commit changes locally
  - [ ] Push to GitHub
  - [ ] Pull on server: `git pull`
  - [ ] Update dependencies if needed

- [ ] **Monitor training**
  - [ ] Check GPU utilization: `nvidia-smi`
  - [ ] Monitor with TensorBoard
  - [ ] Watch for errors in logs

- [ ] **Backup**
  - [ ] Backup trained models
  - [ ] Backup processed data
  - [ ] Backup results

## ‚ö†Ô∏è Common Mistakes to Avoid

- [ ] Don't commit `venv/` or `__pycache__/`
- [ ] Don't commit dataset images
- [ ] Don't commit large model files without Git LFS
- [ ] Don't hard-code server-specific paths
- [ ] Don't commit `.env` files or secrets
- [ ] Don't forget to activate venv before training
- [ ] Don't use `git add -f` to force-add ignored files

## ‚úÖ Verification Steps

After deployment, verify:

- [ ] `nvidia-smi` shows GPU
- [ ] `python -c "import torch; print(torch.cuda.is_available())"` returns `True`
- [ ] All required packages installed: `pip list`
- [ ] Dataset preprocessed correctly
- [ ] Training starts without errors
- [ ] TensorBoard accessible
- [ ] Web GUI launches (if needed)
